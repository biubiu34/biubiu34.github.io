重定向

标准输入、标准输出和标准错误

​ 构建了一个带有编号标记的通道（文件描述符）的进程结构来管理要打开的文件，进程连接到文件，从而到达这些文件所代表的数据内存或设备。通过用户通道0、1、2（称之为标准输入、标准输出和标准错误）的默认连接创建进程。进程使用3以上编号的通道连接 其他文件。

  • 标准的输入 -》键盘–》0
  • 标准的输出 -》显示器 —》正确的 —》1
  • 标准的错误输出 —》显示器 —》错误的 —-》2
  • 其他文件 —-》无 —》读取和/或写入 –》3+

重定向（redirect）

  • 默认情况下，所有的输出都输出到屏幕上，有时候需要改变输出的方向，例如输出到文件里，这就叫做重定向。重新定义输出的方向。

内核

内核的作用—–》强调对进程的管理 –》/proc文件系统

Linux操作系统的基本体系结构

内核的作用：

1.对cpu进程调度管理  ---》cpu
2.对进程进行管理----》进程
3.对内存进行分配管理  ---》memory
4.对文件系统进行管理 ----》disk
5.对其他的硬件进行管理---》sound card、display card 、network card

/proc —一个虚拟文件系统

  • Linux内核提供了一种通过/proc文件系统，在运行时访问内核内部数据结构、改变内核设置的机制

  • /proc文件系统是一种内核和内核模块用来向进程(process)发送信息的机制

  • 与其他文件系统不同，/proc存在于内存（断电数据就消失了）之中而不是硬盘上，所以df 查看不到/proc的信息，关机之后,/proc文件系统里的内容都会消失

  • 查看信息

[root@localhost ~]# cd /proc

  • 每一个进程在/proc文件系统里都有一个蓝色的数字命名的文件夹

文件描述符的介绍

​ 内核—》进程—》文件描述符

  • 文件描述符在形式上是一个非负整数。实际上，它是一个索引值，指向内核为每一个进程所维护的该进程打开文件的记录表。当程序打开一个现有文件或者创建一个新文件时，内核向进程返回一个文件描述符。

  • 文件描述符就是进程打开的每一个文件对应的一个编号，进程通过这个编号来找到相应的文件。

    • 内核帮助进程维护一张文件描述符表：文件名 –》编号

  • 每个进程里面都有自己独立的文件描述符,相互之间没有关系

  • 文件描述符的有效范围是0到OPEN_MAX

    • 调大一个进程最多能打开的文件数量 ulimit -n 65535

      • 相当于对内核参数进行调优

重定向和管道操作相关命令

管道过滤截取相关命令：（echo 、basename、wc 、xargs、 seq 、tr、 cut、 uniq 、sort 、grep）

  • echo

    • 在屏幕上显示一段文字或指定内容

    • 选项

      • -n:使输出和提示符之间没有换行，默认情况下是会换行的

      • -e:是让\后接的字符串有特殊作用。例如\t代表tab键的作用

        • -e支持\t \n \e等

      • \033[31m 31代表红色 32代表绿色

        • \033[0m 关闭颜色显示功能
        • \e 等于\033

  • basename

    • 取出绝对路径里的文件名字

  • 管道操作符号“|”

    • 连接左右两个命令，将左侧的命令的标准输出，作为右侧命令的标准输入
    • 格式：

  • wc

  • xargs

    • 将前面命令的输出送给后面的命令当参数使用

    • 读入stdin的数据转换为参数添加到命令行中

    • 让一些不支持管道的命令可以使用管道

  • seq

    • 打印出一串有序的数字【序列】

    • 格式 :seq [选项] 数字范围

    • 选项

      • -s 指定分隔符

        • 默认情况下分隔符是回车\n

      • -w :指定同等宽带输出

  • $()

    • 让命令在里面执行，得到这个命令的执行的结果

  • tr

    • 字符转换命令

    • 只能对stdin操作，而不能直接对文件操作

    • 使用： tr set1 set2

      • 用set2中的字符替换掉set1 中同一位置的字符

  • cut

    • 从文本文件或文本刘中提取文本列

    • cut -选项 提取范围 文本文件

    • 常见选项

      • -c :从指定提取范围中提取字符
      • -f :从指定提取范围中提取字段
        • -d：指定分隔符，默认分隔符是TAB键

    • t提取范围

      • n 第n项
      • n-:第n项到行尾
      • -m :行首到第m项
      • n,m：第n项和第m项
      • n-m:第n-m项

  • uniq

    • 删除经过排序后的数据的重复记录（去重）
    • 经常和sort 连用，先排序，再去重
      • sort -n tt|uniq
    • 选项
      • -c:统计特定记录出现的次数
      • -u :只显示唯一的行
      • -d 只显示重复的行

  • sort

    • 默认按每行的第一个字符排序，比较首字母的ASCII编码值的大小来排序（默认升序）
    • 选项：
      • -n :按整数进行排序
      • -r 递减排序（默认是升序，-r是降序）
      • -k:指定哪一列为排序键
        -
      • -t :指定字段分隔符（默认是空白）

  • grep

    • 在文件中查找并显示包含指定字符串的行，正则表达式，查找过滤

    • 工具：根据用户给的正则表达式去文件里搜索符号符合的内容，然后将内容在屏幕是上显示，默认情况下，将符号要求的内容所在的行显示在屏幕是上

    • grep 是一个文本过滤命令，根据条件来进行过滤，符号要求的行在屏幕上显示。

      • 条件 =通过一个 模式来表示
      • 模式里可以一个正则表达式也可以是字符串

    • 格式 grep 【选项】…模式 目标文件

    • 元字符：有特殊作用的字符，表达特殊的含义

    • 基本正则：

      • 基本正则：特殊字符（元字符）比较少 —–》grep支持基本正则
      • 扩展正则：元字符比较多，新加入了一些进来 —》egrep支持扩展正则

    • 选项

      • -i :查找时候忽 略大小写

      • -v:反转查找，输出与模式不相符的行

      • -o:仅仅显示符合匹配的内容

      • -n: 显示符合模式要求的行号

      • -r:递归搜索所有的文件

      • -a :以文本文件方式搜索

      • –color :将匹配的内容用红色表示，建议使用

      • -An:同时显示符合条件行的下面n行 A:after

      • Bn：同时显示符合条件行的上面n行 B:before

      • -Cn:同时显示符合条件行的上面和下面n行

      • -E：支持扩展正则表达式 grep -E =egrep

    • 模式

      • ^……：以…开头

      • ……$:以…结尾

      • ^$:表示空行

      • .表示任意单个字符

      • \bwubing\b 界定单词，只是查找wubing单词，包含wubing 的不算

      • \bwubng :单词以wubing 开头

      • wubing\b ：单词以wubing结尾

      • \\ :界定单词，只是查找wubing单词，包含wubing的不算

      • grep :在定界单词的时候，默认使用的分隔符是空白（空格和tab）,其他的符号也可以作为单词的定界符号。

      • \：续航符号

      • \转义字符：让元字符没有特殊的含义，恢复到字符本身的意思

  • egrep

    • egrep = grep -E

  • wc

    • 字数统计命令

    • 格式 ：wc 【选项】…目标文件…

    • 选项

      • -l :统计行数

      • -w :统计字数（前后都是空白的一组字符）

      • -c 统计字符数（可见和不可见的字符）

  • tee

    • 是实现重定向和同时也在屏幕上输出的功能

扩展重定向相关命令：(diff ,patch)

  • diff
    • 比较两个文件之间的差异，输出结果为两个文件的不同之处
    • 输出格式
      • 标准diff
      • -u :会将不同的地方放在一起，紧凑易读
      • -r :递归比较目录下的所有文件
    • 利用diff命令生成补丁
      • diff -u test1 test2 >test.patch
  • patch
    • 用途：用来打补丁修复文件
    • 格式：patch 【选项】 原始文件 <补丁文件
    • -pN :N表示忽略N层路径
    • -R:回滚还原到老版本
    • 注意事项
      • 如果打多个补丁，注意先后顺序
      • 打补丁前不要修改源文件
    • 顺序
      • 1.生成补丁文件
      • 2.打补丁（旧的文件）
      • 3.可以回滚带旧版本
    • 目录补丁生成和打补丁
      • 1.生成补丁
        • diff -Nur QQv1 QQv2 >v1-to-v2.patch
        • -r :递归的比较
        • -N 没有的文件，另一方会拿一个空文件和它进行对比
      • 2.打补丁（旧的文件）
      • 3.可以回滚到就版本
