重定向

标准输入、标准输出和标准错误

​ 构建了一个带有编号标记的通道（文件描述符）的进程结构来管理要打开的文件，进程连接到文件，从而到达这些文件所代表的数据内存或设备。通过用户通道0、1、2（称之为标准输入、标准输出和标准错误）的默认连接创建进程。进程使用3以上编号的通道连接 其他文件。

  • 标准的输入 -》键盘–》0
  • 标准的输出 -》显示器 —》正确的 —》1
  • 标准的错误输出 —》显示器 —》错误的 —-》2
  • 其他文件 —-》无 —》读取和/或写入 –》3+

重定向（redirect）

  • 默认情况下，所有的输出都输出到屏幕上，有时候需要改变输出的方向，例如输出到文件里，这就叫做重定向。重新定义输出的方向。

内核

内核的作用—–》强调对进程的管理 –》/proc文件系统

Linux操作系统的基本体系结构

内核的作用：

1.对cpu进程调度管理  ---》cpu
2.对进程进行管理----》进程
3.对内存进行分配管理  ---》memory
4.对文件系统进行管理 ----》disk
5.对其他的硬件进行管理---》sound card、display card 、network card

如何查看进程

1、ps aux :

  • 查看某一个瞬间的系统中运行的所有的进程信息

  • pid 就是进程的进程号

2.top 动态查看进程的信息

  • 使用top命令查看进程的时候
    • 默认情况下，按cpu的使用率排序
    • shift+> 下一页进程信息
    • shift +< 上一页进程信息
    • M 是按照内存的使用率进行排序
    • q 是退出

/pro目录文件系统

文件系统：

  • 其实就是磁盘上简历的一种管理文件的方法，具体事情例如：
    • 文件的读取、写入
    • 文件命名
    • 文件最大可以多大
    • 对文件的复制、删除、重命名等操作
    • 一个文件夹里可以包含多少个文件
    • 一个分区到底可以多大？
    • ….

/proc —一个虚拟文件系统

  • Linux内核提供了一种通过/proc文件系统，在运行时访问内核内部数据结构、改变内核设置的机制

  • /proc文件系统是一种内核和内核模块用来向进程(process)发送信息的机制

  • 与其他文件系统不同，/proc存在于内存（断电数据就消失了）之中而不是硬盘上，所以df 查看不到/proc的信息，关机之后,/proc文件系统里的内容都会消失

  • 查看信息

[root@localhost ~]# cd /proc

  • 每一个进程在/proc文件系统里都有一个蓝色的数字命名的文件夹

文件描述符的介绍

​ 内核—》进程—》文件描述符

  • 文件描述符在形式上是一个非负整数。实际上，它是一个索引值，指向内核为每一个进程所维护的该进程打开文件的记录表。当程序打开一个现有文件或者创建一个新文件时，内核向进程返回一个文件描述符。

  • 文件描述符就是进程打开的每一个文件对应的一个编号，进程通过这个编号来找到相应的文件。

    • 内核帮助进程维护一张文件描述符表：文件名 –》编号

  • 每个进程里面都有自己独立的文件描述符,相互之间没有关系

  • 文件描述符的有效范围是0到OPEN_MAX

    • 调大一个进程最多能打开的文件数量 ulimit -n 65535

      • 相当于对内核参数进行调优

重定向和管道操作相关命令

管道过滤截取相关命令：（echo 、basename、wc 、xargs、 seq 、tr、 cut、 uniq 、sort 、grep）

  • echo

    • 在屏幕上显示一段文字或指定内容

    • 选项

      • -n:使输出和提示符之间没有换行，默认情况下是会换行的

      • -e:是让\后接的字符串有特殊作用。例如\t代表tab键的作用

        • -e支持\t \n \e等

      • \033[31m 31代表红色 32代表绿色

        • \033[0m 关闭颜色显示功能
        • \e 等于\033

  • basename

    • 取出绝对路径里的文件名字

  • 管道操作符号“|”

    • 连接左右两个命令，将左侧的命令的标准输出，作为右侧命令的标准输入
    • 格式：

  • wc

  • xargs

    • 将前面命令的输出送给后面的命令当参数使用

    • 读入stdin的数据转换为参数添加到命令行中

    • 让一些不支持管道的命令可以使用管道

  • seq

    • 打印出一串有序的数字【序列】

    • 格式 :seq [选项] 数字范围

    • 选项

      • -s 指定分隔符

        • 默认情况下分隔符是回车\n

      • -w :指定同等宽带输出

  • $()

    • 让命令在里面执行，得到这个命令的执行的结果

  • tr

    • 字符转换命令

    • 只能对stdin操作，而不能直接对文件操作

    • 使用： tr set1 set2

      • 用set2中的字符替换掉set1 中同一位置的字符

  • cut

  • uniq

  • sort

  • grep
