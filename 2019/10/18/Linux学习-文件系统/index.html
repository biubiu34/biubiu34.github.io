  • 管理磁盘分区

    • 主分区可以引导操作系统

    • 基本磁盘上可以创建一到四个主分区

      • 主分区数目在MBR中记录

    • 磁盘分区需要超过4个时

      • 可将剩余空间作为1个扩展分区
      • 扩展分区会占用- -个主分区位置

    • 扩展分区不能直接存储文件

      • 划分成逻辑分区来存储文件
      • 1个扩展分区可以划分成多个逻辑分区

    • 分区类型

      • 主分区 primary partition
        • 可以存放数据、安装系统、有可以存放资料
        • 分区编号1-4
      • 扩展分区extend partition：
        • 最多一个。用来图片4个分区的限制，不能进行格式化，只是一个概念
        • 扩展分区占一个主分区的位置
        • 逻辑分区 logical partition ：
          • 用来存放数据的
          • logical partition 分区编号从5开始

    • 分区表（parition table）

      • 记录一块磁盘里有多少个分区的表，分区的信息记录在此。 parition table

      • DOS类型

        • 最多4个主分区

      • GPT类型

        • 不受限制，可以很多主分区

    • 磁盘和磁盘分区的关系

      • 分区是从磁盘里划分出来的一块空间。

      • 新买的硬盘第一次使用的时候，需要进行分区

      • 为什么要分区？

        方便分类存放，查找数据方便---》人

文件系统

  • 含义

    • 文件系统:对分区里的文件进打管理的系统，例如文件的命名文件读取，文件的查找等行为进行管理
    • windows:NTF,FAT32
      • 建议使用NTF 支持分区和大文件
      • FAT32文件系统不支持超过4G的文件
    • ​ centos7里：xfs

  • 文件系统的种类

    • Linux下的文件系统

      1. XFS —》centos7
      2. ext4 —》centos6
      3. ext3 –》centos5
      4. iso9660 光盘
      5. tmpfs –》内存里划分一块空间，使用这种文件系统

      1. NTFS –》new technology file system
      2. FAT32 —》u盘

      2. GFS

    • 实现自动挂载

      • /dev/sdb6 表示设备文件—》分区

        • 使用设备名称（/dev/sda)来挂载分区时是被固定死的，一旦磁盘的插槽顺序发生了变化，就会出现名称不对应的问题。因为这个名称是会改变的。

        /games 挂载点

        • 就是你要挂载到哪个目录下。

          xfs 文件系统的格式

        • 磁盘文件系统的格式，包括ext2、ext3、reiserfs、nfs、vfat等

          defaults 挂载的时候采用的选项，使用默认选项–》rw（可读可写）

        • Async/sync 设置是否为同步方式运行，默认为async
          auto/noauto 当下载mount -a 的命令时，此文件系统是否被主动挂载。默认为auto
          rw/ro 是否以以只读或者读写模式挂载
          exec/noexec 限制此文件系统内是否能够进行”执行”的操作
          user/nouser 是否允许用户使用mount命令挂载
          suid/nosuid 是否允许SUID的存在
          Usrquota 启动文件系统支持磁盘配额模式
          Grpquota 启动文件系统对群组磁盘配额模式的支持
          Defaults 同事具有rw,suid,dev,exec,auto,nouser,async等默认参数的设置

          0 对dump的支持，是否可以去备份分区 0 不备份

        • 0 代表不要做dump备份
          1 代表要每天进行dump的操作
          2 代表不定日期的进行dump操作

          0 开机启动的时候，是否去分区的文件系统进行检测扇区 0 不检测

        • 开机的过程中，系统默认会以fsck检验我们系统是否为完整（clean）

        • 0 不要检验
          1 最早检验（一般根目录会选择）
          2 1级别检验完成之后进行检验

    • 使设置生效

      • 重启
        • reboot
        • init 6
        • shutdown -r now
      • 关机
        • init 0
        • halt
        • shutdown -h now

    • 重启之后使用df -Th看是否生效

    • /etc/fstab文件的作用

      • 磁盘被手动挂载之后都必须把挂载信息写入/etc/fstab这个文件中，否则下次开机启动时仍然需要重新挂载。

      • 系统开机时会主动读取/etc/fstab这个文件中的内容，根据文件里面的配置挂载磁盘。这样我们只需要将磁盘的挂载信息写入这个文件中我们就不需要每次开机启动之后手动进行挂载了。

      • 挂载的限制

        • 　1、根目录是必须挂载的，而且一定要先于其他mount point被挂载。因为mount是所有目录的跟目录，其他木有都是由根目录 /衍生出来的。

          2、挂载点必须是已经存在的目录。

          3 、挂载点的指定可以任意，但必须遵守必要的系统目录架构原则4、所有挂载点在同一时间只能被挂载一次

          5、所有分区在同一时间只能挂在一次

          6、若进行卸载，必须将工作目录退出挂载点（及其子目录）之外。

  • MBR

    • MBR是什么

      • Master Boot Record：主引导记录–》用来启动我们的操作系统。

      • 硬盘的0柱面、0磁头、1扇区称为主引导扇区 –》只占1个扇区 –》512字节

      • MBR大小512字节，分为三个部分

        主引导程序：446字节 –》启动系统的
        硬盘分区表DPT：64字节 –》所有的分区信息存放在分区表里
        分区结束标记：2字节

        MBR不属于任何一个分区，是独立的空间，是每块磁盘最重要的部分，不能出问题

    • DPT：磁盘分区表

      • DPT大小64字节
        每个主分区要用16个字节描述
        扩展分区也要占用16个字节的主分区空间
